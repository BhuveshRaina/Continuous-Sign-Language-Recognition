## 1. Introduction
**a. Brief description of the problem**
Continuous Sign Language (CSL) recognition is a complex challenge in computer vision and NLP. Unlike isolated sign recognition, continuous signing involves fluid transitions, co-articulation, and varying speeds. The problem requires translating a sequence of video frames containing human gestures into a coherent text sentence. 

**b. Objective of the model**
The primary objective is to design a deep learning model that takes a preprocessed CSL video (represented as keypoint coordinates) as input, extracts spatial and temporal features, and accurately predicts the correct sentence label out of 108 canonical classes.

---

## 2. Data Preparation for Modeling
**a. How videos were loaded**
Instead of loading raw video pixels, the MediaPipe framework was used to extract skeletal landmarks (Pose, Left Hand, Right Hand) from each video. These landmarks were flattened into a 225-dimensional feature vector per frame and saved as `.npy` files to optimize I/O performance during training.

**b. Frame extraction strategy**
Videos naturally vary in duration. To standardize the input size for batch processing, a maximum sequence length of `MAX_FRAMES = 60` was established. 
- **Padding:** Sequences shorter than 60 frames were padded with zeros at the end.
- **Truncating:** Sequences longer than 60 frames were truncated to the first 60 frames.

**c. Preprocessing steps**
1. **Label Normalization:** The initial dataset contained 197 noisy variations of labels (e.g., "I dont agree" vs "I do not agree"). A text preprocessing pipeline merged these into 108 canonical classes, achieving a 45% reduction in redundancy.
2. **Data Cleaning:** NaNs generated by MediaPipe tracking failures were converted to zeros.
3. **Data Augmentation:** During training, a 0.5% Gaussian spatial noise was added to the keypoint tensors to simulate natural variations in human body size and camera angles, preventing overfitting.

**d. Dataset split used for training/testing**
The dataset was randomly split ensuring all 108 classes were represented:
- **Training Set:** 70% (1,658 videos)
- **Validation Set:** 15% (321 videos)
- **Testing Set:** 15% (432 videos)

## 3. Model Architecture
**a. Description of the model used**
The model utilizes a **Bidirectional Long Short-Term Memory (Bi-LSTM) Network with Temporal Attention**. 

**b. Architecture Diagram (Text Representation)**
`Input (Batch, 60, 225) -> Feature Extractor -> Bi-LSTM -> Temporal Attention -> Classifier -> Output (Batch, 108)`

**c. Explanation of each component**
- **Feature Extractor:** A multi-layer perceptron (Linear -> LayerNorm -> GELU -> Dropout) that projects the raw 225 keypoints into a richer 256-dimensional feature space.
- **Bi-LSTM:** Processes the sequence in both forward and reverse directions to capture temporal flow and context, outputting a 512-dimensional hidden state per frame.
- **Temporal Attention:** Instead of simple mean-pooling, an attention mechanism assigns learned weights to each frame, allowing the model to focus on the active gestures and ignore static padded frames.
- **Classifier:** A fully connected network that maps the aggregated temporal features to the 108 final sentence classes.

**d. Reason for choosing the architecture**
A Bi-LSTM was chosen over standard CNNs because CSL is inherently a time-series problem where the trajectory of motion dictates meaning. Bidirectionality ensures the model understands the context of a gesture based on what comes before and after it. Temporal Attention was added to handle the zero-padded frames effectively.

---

## 4. Training Setup
**a. Loss function**
**CrossEntropyLoss** with a Label Smoothing factor of 0.1 was utilized. Label smoothing prevents the model from becoming overly confident, which acts as a regularizer for our relatively small dataset.

**b. Batch size**
A batch size of **64** was used to maximize GPU utilization while maintaining stable gradients.

**c. Number of epochs**
The model was trained for **100 epochs**. An Early Stopping mechanism and model checkpointing were implemented to save the weights that produced the highest validation accuracy.

**d. Hardware/software used**
- **Hardware:** Google Colab T4 GPU
- **Software:** PyTorch 2.x, Python 3.10
- **Optimizations:** Automatic Mixed Precision (AMP) for faster training, AdamW Optimizer with Weight Decay (1e-2), and Cosine Annealing Learning Rate Scheduler.

## 6. Discussion and Limitations
**a. Observations from results**
The model learned the training data exceptionally well (99.6% accuracy) and converged rapidly within the first 20 epochs. A validation accuracy of 68.5% across 108 distinct classes is a strong result, demonstrating that skeletal keypoints contain sufficient signal for sentence-level recognition.

**b. Errors and failure cases**
Reviewing the failure cases, the model primarily confuses sentences that share highly similar initial hand trajectories or occur in similar contexts (e.g., confusing "can i help you" with "bring water for me"). It also struggles with classes that have very slight morphological differences.

**c. Limitations of the model**
The primary limitation is **Overfitting**, evident by the gap between training (~99%) and validation (~68%) accuracy. This is due to the limited number of samples per class (~15-20). Additionally, because facial expression landmarks were not deeply emphasized, non-manual markers (like raised eyebrows for questions) might be ignored by the model.

**d. Possible improvements**
1. **Dataset Expansion:** Collecting more diverse variations of each sign.
2. **Architecture Upgrade:** Migrating from Bi-LSTM to a Spatio-Temporal Graph Convolutional Network (ST-GCN) or a Transformer Encoder to capture spatial relationships between joints better.
3. **Advanced Augmentation:** Implementing Time-Warping or Frame Masking to make the model more robust to different signing speeds.
